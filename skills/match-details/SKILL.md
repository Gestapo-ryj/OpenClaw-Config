# Match Details Skill

## Purpose
通过球队名称搜索球队ID，获取球队赛程，然后获取具体比赛的详细数据。特别适用于titan007.com体育数据平台。

## When to Use
- 用户询问特定球队的比赛详情
- 需要获取球队的完整赛程信息
- 需要分析具体比赛的详细统计数据
- 需要获取比赛阵容、进球时间线、技术统计等详细信息

## Input Requirements
1. **球队名称**：需要查询的球队中文名称（如"华奇巴托"）
2. **可选参数**：
   - 比赛ID（如果已知）
   - 时间范围（如最近比赛、特定日期）
   - 联赛筛选

## Workflow

### 步骤1：球队搜索
1. 访问 `https://ba2.titan007.com/homepage/multisearch?keyword=[球队名称]&type=0`
2. 解析API响应，提取球队ID
3. 验证搜索结果，确保找到正确的球队

### 步骤2：获取球队信息
1. 访问球队详情页面：`https://zq.titan007.com/cn/team/Summary/[teamId].html`
2. 获取球队基本信息（名称、联赛、教练等）
3. 提取球队数据文件链接（如tdl[teamId].js）

### 步骤3：获取球队赛程
1. 访问赛程页面：`https://zq.titan007.com/cn/team/TeamSche/[teamId].html`
2. 解析JavaScript数据文件获取完整赛程
3. 提取比赛列表（时间、对手、比分、比赛ID）

### 步骤4：获取比赛详情
1. 对于特定比赛，访问详情页面：`https://live.titan007.com/detail/[matchId]cn.htm`
2. 解析比赛数据：
   - 技术统计（射门、控球、角球等）
   - 阵容信息（首发、替补）
   - 进球时间线
   - 比赛事件（换人、黄牌等）
   - 球队近期表现统计

### 步骤5：数据分析和报告生成
1. 分析比赛关键数据
2. 识别比赛趋势和模式
3. 生成易读的综合报告

## Configuration

### 默认参数
```yaml
teamName: ""  # 球队名称（必需）
matchId: ""   # 比赛ID（可选，如果提供则直接获取比赛详情）
timeRange: "recent"  # 时间范围：recent/all/upcoming
limit: 10     # 显示比赛数量限制
detailLevel: "full"  # 详情级别：summary/full
```

### 可调整参数
- `timeRange`: 时间范围（最近比赛/全部比赛/未来比赛）
- `limit`: 显示的比赛数量
- `detailLevel`: 详情详细程度
- `includeStats`: 是否包含技术统计
- `includeLineup`: 是否包含阵容信息
- `includeEvents`: 是否包含比赛事件

## Error Handling

### 常见错误
1. **球队未找到**: 检查球队名称拼写，尝试别名搜索
2. **页面结构变化**: 调整解析逻辑，使用备用选择器
3. **网络问题**: 重试机制，超时处理
4. **数据不完整**: 标记缺失数据，继续处理可用部分

### 降级方案
1. 使用简化数据提取（如仅基本比分）
2. 提供部分数据而非完全失败
3. 提示用户手动检查特定页面

### 数据流优化
```javascript
// 完整的数据获取流程
async function getCompleteMatchAnalysis(teamName, matchId = null) {
  // 1. 搜索球队
  const teamInfo = await searchTeam(teamName);
  
  // 2. 获取球队赛程
  const schedule = await getTeamSchedule(teamInfo.id);
  
  // 3. 确定目标比赛

  // 5. 生成报告
  return generateReport(teamInfo, schedule, matchDetails);
}
```

## Best Practices

### 数据获取建议
1. **使用适当请求头**: 添加浏览器头信息避免被屏蔽
2. **处理JavaScript内容**: 对于动态加载的数据，可能需要分析JS文件
3. **数据验证**: 验证获取的数据完整性和准确性
4. **缓存策略**: 对于频繁查询的数据考虑缓存

### 分析建议
1. **关注关键指标**: 射正率、预期进球、控球率等
2. **结合上下文**: 考虑球队状态、伤病、历史交锋等因素
3. **趋势分析**: 分析球队近期表现趋势
4. **对比分析**: 与对手和联赛平均水平对比

### 呈现建议
1. **结构化输出**: 使用清晰的标题和表格
2. **突出重点**: 用符号和颜色标记重要信息
3. **提供上下文**: 解释数据的意义和影响
4. **保持客观**: 基于数据提供分析，避免主观判断
